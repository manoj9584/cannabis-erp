🌿 Cultivation Module
Complete cultivation transaction model in six layers: IAM & multi-tenancy (tenant → facility → user role assignments with granular CRUD permissions), Infrastructure (building → room → zone → table with live environmental logging), Genetics chain (genetics registry → inventory → propagation batches → cultivation lots), Lot lifecycle (stage logs, zone placements with inline env data, transplant events, care tasks), Agronomic inputs (external feeds, substrate assignments), and Harvest chain (pre-harvest check → wet weight → drying → post-harvest batch → QC tests). Destruction records span genetics, propagation, and lot stages.

🔑
CULTIVATION_LOT is the central write target — receives FK references from placement, care, feeds, substrates, transplant events, and the entire harvest chain. One record per propagation batch per phase.
📍
LOT_PLACEMENT carries inline environment readings (ec_value, ph_value, runoff_volume_ml, temp, humidity, light_intensity) — no separate join needed for spot environmental context at a placement event.
💀
DESTRUCTION_RECORD is a multi-source polymorphic table — nullable FKs to genetic_inventory_id, propagation_batch_id, and cultivation_lot_id with destruction_stage enum controlling which is populated.

📦 Inventory, Cultivator Stock & Sales Module
Five-flow full cultivator inventory model.   Genetics Stock — GENETICS registry → GENETIC_INVENTORY (mothers, seeds, clones, teens) with on-hand count, acquisition source, and depletion tracking → in-progress PROPAGATION_BATCH clones → CULTIVATION_LOT live plant counts by phase. All four are cross-module references (†) providing real-time pre-production visibility.  Input Stock — CULTIVATION_INPUT catalog items tracked via INPUT_STOCK (on-hand qty, par level, storage location) with every replenishment and consumption logged in INPUT_MOVEMENT. PO receipts increment INPUT_STOCK; lot feed applications decrement it.  Finished Goods — POST_HARVEST_BATCH → PACKAGE → INVENTORY_ITEM with grade, Aw, and qty depletion.  Procurement — VENDOR → SUPPLY_CATALOG → PURCHASE_ORDER → PO_LINE covering all non-cultivation-input supplies.  Sales — CUSTOMER → SALES_ORDER → SALES_LINE with full compliance + METRC fields.

🧬
GENETIC_INVENTORY is now a visible inventory entity (not just a Module 1 concern). available_quantity tracks mothers, seeds, clones, and teens on hand. PROPAGATION_BATCH and CULTIVATION_LOT provide in-progress and live plant counts to complete the pre-production inventory picture.
🧪
INPUT_STOCK is the new real-time on-hand tracker for cultivation inputs (nutrients, substrates, IPM). qty_on_hand increments on INPUT_MOVEMENT type RECEIPT (PO delivery) and decrements on CONSUMPTION (lot feed application). par_level breach triggers reorder.
🏦
INVENTORY_ITEM.qty_available is the atomic finished-goods depletion counter — decrements on SALES_LINE write, increments on return. Cross-module bridge: PACKAGE.post_harvest_batch_id → POST_HARVEST_BATCH.
🚚
CUSTOMER carries all CA distribution compliance fields: state license, distro license, driver name/DL/phone, vehicle plate — required for METRC transfer manifests. metrc_transfer_id on SALES_ORDER ties to the filed manifest.
